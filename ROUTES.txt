═══════════════════════════════════════════════
  PUBLIC ROUTES (No Auth Required)
═══════════════════════════════════════════════

GET  /                → Analytics Dashboard (HTML UI)
GET  /health          → Health Check ("BFOBS is Online 🧠🚀")
GET  /api/status      → Real-time JSON analytics (keys, logs, timeline)

═══════════════════════════════════════════════
  DEVELOPER PROXY GATEWAY (Product Auth Required)
  Header: Authorization: Bearer <Product_OAuth_Token>
═══════════════════════════════════════════════

POST /api/db/p/:product_slug/execute  → Execute SQL on product shards
                                        Body: { "query": "...", "params": [...] }
                                        Logic: Auto-routes to correct shard fleet.

═══════════════════════════════════════════════
  ADMIN INFRASTRUCTURE (Master Key Required)
  Header: Authorization: Bearer <BANDHANNOVA_MASTER_KEY>
═══════════════════════════════════════════════

GET  /api/admin/db/status         → List all shards and their health
POST /api/admin/db/provision      → Link/Register a new manual shard
PUT  /api/admin/db/update/:id      → Update shard credentials/URL
POST /api/admin/db/remove/:id     → Decommission/Remove a shard (Requires DELETE text)
POST /api/admin/db/execute        → Run SQL on a specific shard ID
POST /api/admin/db/execute-bulk   → Run SQL across ALL shards of a product
GET  /api/admin/products          → List all managed infrastructure products
POST /api/admin/products          → Register new product infrastructure
